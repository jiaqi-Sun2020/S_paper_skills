# Claim Safety

Use this reference before any rewrite.

## Non-Negotiable Rules

- Keep completed, partial, planned, and speculative work distinct.
- Preserve all quantitative values unless the user asks for a correction or an obvious formatting fix.
- Do not turn an implementation detail into a scientific claim.
- Do not turn a visualization trend into statistical significance.
- Do not invent citations, priority statements, reviewer comparisons, or baselines.
- If a result is based on a subset of experiments, state the subset.

## Claim Ladder

Use the lowest defensible level:

1. `We propose` for method or framework introduction.
2. `We implement` for verified code or pipeline construction.
3. `We observe` for reported experimental behavior.
4. `We find` for results supported by the present analysis.
5. `We show` for claims supported by targeted evidence.
6. `We demonstrate` only when evidence is complete and directly tied to the claim.

## Revision Notes

If a rewrite softens or narrows a claim, state that in `Revision notes:`. If a claim cannot be fixed without new data, put it under `Needs author check:`.
